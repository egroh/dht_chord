3//! This repository showcases a Rust P2P-DHT-Chord implementation.
4//!
5//! It was designed with a focus on performance, security and reliability.
6//! The entire project is heavily multithreaded and asynchronous.
7//! It features built-in replication of content,
8//! as well as a proof-of-work challenge to prevent Sybil attacks.
9//! Node discovery and network stabilization are performed automatically
10//! whenever new nodes join or leave the network.
11//!
12//! It was created as part of the [Peer-to-Peer Systems and Security](https://www.net.in.tum.de/teaching/ss23/p2p.html) lecture at TUM.
13//!
14//!
15//! # Architecture:
16//! All you need to build our package is the latest (nightly) version of the Rust compiler and Cargo.
17//!
18//! Installation instructions for Rust are provided [here](https://www.rust-lang.org/tools/install).
19//!
20//! For reference, we are currently working with the following versions:
21//! - `cargo 1.74.0-nightly (96fe1c9e1 2023-08-29)`
22//! - `rustc 1.74.0-nightly (35e416303 2023-09-01)`
23//!
24//! [This documentation](https://valentin-metz.github.io/dht_chord/dht_chord/index.html) was built with:
25//! - `rustdoc 1.74.0-nightly (35e416303 2023-09-01)`
26//!
27//! # Main executable:
28//! The main executable of this program `p2p_dht` takes one command line argument, `-c <config>`,
29//! where `<config>` is the path to a configuration file.
30//!
31//! The configuration file is expected to be in the INI format. The following configuration options are available:
32//! ```ini
33//! [dht]
34//! default_store_duration = 60     ; Default storage duration in seconds
35//! max_store_duration = 600        ; Maximum storage duration in seconds
36//! p2p_address = 127.0.0.1:8001    ; Address to listen on for P2P connections
37//! api_address = 127.0.0.1:7401    ; Address to listen on for API connections
38//! bootstrap_node = 127.0.0.1:8000 ; (Optional) Address of a bootstrap node
39//! ```
40//! To run our program, you may use:
41//! ```bash
42//! cargo run --release -- -c example_config_node_0.ini
43//! ```
44//! We support different levels of logging, with the default being `info`.
45//! You can change the log level by setting the `RUST_LOG` environment variable to one of the following values:
46//! - `error`
47//! - `warn`
48//! - `info`
49//! - `debug`
50//! - `trace`
51//!
52//! The latest `x86_64-unknown-linux-gnu` binary of our program is provided as an artifact on our [GitHub repository](https://github.com/Valentin-Metz/dht_chord/actions/workflows/rust.yml).
53//!
54//! Alternatively, you may choose to build our program yourself:
55//! ## Compilation:
56//! After cloning this repository, you may run the following command to build our program:
57//! ```bash
58//! cargo build --release
59//! ```
60//! This will create a binary in `target/release/dht_chord`.
61//!
62//! You can also build this documentation locally:
63//! ```bash
64//! cargo doc --no-deps --document-private-items
65//! ```
66//! The documentation will be created in `target/doc/dht_chord/index.html`.
67//! ## Testing:
68//! We provide a CI/CD pipeline that runs our tests on every commit.
69//! You may find the latest results [here](https://github.com/Valentin-Metz/dht_chord/actions).
70//!
71//! Alternatively, you can run our test suit yourself:
72//! ```bash
73//! cargo test --release
74//! ```
